# ShopSimulator 项目计划书

---

## 一、项目概述

### 1.1 项目名称
**ShopSimulator** —— 极简模拟购物平台

### 1.2 项目定位
一个自娱自乐的模拟购物平台，以**极简主义 + 极客审美**为核心设计语言。用户可扮演管理员、商户、买家三种角色，在简洁高效的界面中自由创建商品并完成交易，体验“买买买”的纯粹快感。

### 1.3 核心价值
- 无需真实货币，纯粹体验购物满足感
- 高度自由的商品与商户自定义
- 极简界面，专注操作效率，零干扰

### 1.4 目标用户
追求极简操作体验、热衷自娱自乐的个人用户。

## 二、视觉与交互设计

### 2.1 整体风格关键词
**极简主义 × 极客审美**

### 2.2 色彩方案

| 用途 | 色值 | 说明 |
|------|------|------|
| 主背景 | `#0D0D0D` | 深黑底色（暗色默认主题） |
| 卡片/面板 | `#1A1A1A` | 微弱抬升的灰黑 |
| 边框/分割线 | `#2A2A2A` | 极低调的分隔 |
| 主文字 | `#E5E5E5` | 柔和白色，不刺眼 |
| 次要文字 | `#888888` | 中灰，辅助信息 |
| **强调色** | `#00FF88`（荧光绿） | 按钮、链接、选中态 |
| 辅助强调色 | `#FF6B35`（亮橙） | 警告/删除/金额提示 |
| 成功色 | `#00FF88` | 操作成功反馈 |
| 错误色 | `#FF4444` | 操作失败/警告 |

### 2.3 字体系统

| 用途 | 字体 | 备选 | 说明 |
|------|------|------|------|
| 代码/数据 | `JetBrains Mono` | `Consolas`, `monospace` | 等宽字体，数字对齐 |
| 标题 | `Inter` | `SF Pro Display`, `sans-serif` | 几何无衬线体 |
| 正文 | `Inter` | 同上 | 清爽可读 |

### 2.4 布局原则
- **留白充裕**：内容之间大量留白，视觉呼吸感强烈
- **卡片式布局**：所有模块以微弱圆角卡片承载
- **去除冗余**：无线条、无阴影、无渐变，仅凭明度差区分层级
- **栅格系统**：基于 8px 栅格，间距统一为 8 的倍数

### 2.5 动效规范
- **原则**：微交互为主，服务于功能，不喧宾夺主
- **hover**：150ms ease-out 过渡，仅做轻微亮度变化
- **点击**：瞬间反馈（背景色翻转）
- **路由切换**：无过渡动画，直接切换（追求速度感）
- **数据加载**：骨架屏（skeleton）代替传统 spinner

### 2.6 设计参考
| 参考对象 | 借鉴要点 |
|----------|----------|
| **Apple 官网** | 极致留白、排版节奏、内容呼吸感 |
| **GitHub 暗色模式** | 暗色极客氛围、代码感界面 |
| **Linear** | 精致的微交互、键盘快捷键体验 |

## 三、技术架构

### 3.1 技术栈选定

| 层级 | 技术 | 选型理由 |
|------|------|----------|
| 前端 | React 18 + Vite | 极速开发体验，契合“极客”调性 |
| 路由 | React Router v6 | 页面级路由管理 |
| UI 框架 | **无**，手写 CSS/Tailwind | 追求极致定制，不引入组件库臃肿样式 |
| 样式方案 | Tailwind CSS + CSS Variables | 快速构建，变量管理色彩主题 |
| HTTP 请求 | Axios | 简洁的 API 调用 |
| 状态管理 | Zustand | 轻量、无模板代码，符合极简哲学 |
| 后端 | Node.js + Express | JavaScript 统一前后端语言 |
| 数据库 | MySQL | 结构化数据，事务支持 |
| ORM | Prisma | 类型安全、迁移简洁 |
| 认证 | JWT + bcrypt | 无状态认证，安全可靠 |

## 四、功能模块拆解

### 4.1 模块划分

```
ShopSimulator
├── 认证模块
│   ├── 注册（选择角色：商户 / 用户）
│   ├── 登录
│   └── JWT 持久化
├── 管理员模块
│   ├── 用户管理（列表、编辑、删除）
│   ├── 商户管理（列表、编辑、删除）
│   ├── 订单管理（查看全部、删除）
│   └── 打钱（指定用户 + 金额）
├── 商户模块
│   ├── 商品管理（上架、编辑、下架、删除）
│   ├── 销售记录查看
│   └── 余额查看
├── 用户模块
│   ├── 店铺浏览（首页）
│   ├── 商品浏览（店铺详情页）
│   ├── 购买商品（扣款 + 生成订单）
│   ├── 购买记录查看
│   └── 余额查看
└── 全局
    ├── 暗黑/亮色模式切换
    ├── 路由守卫（按角色）
    └── 骨架屏加载状态
```

### 4.2 页面 UI 描述（极客审美体现）

| 页面 | 视觉要点 |
|------|----------|
| **登录/注册** | 居中单一表单，深黑背景，荧光绿输入框聚焦边框，无任何装饰 |
| **首页** | 2 列网格卡片布局展示店铺，卡片仅含店名 + 简介，hover 时边框变荧光绿 |
| **店铺详情页** | 商品列表，等宽字体显示价格（右对齐），购买按钮极简只有 `+` 号 |
| **管理后台** | 数据表格风格，斑马纹微弱区分行，操作按钮仅为文字链接（无底色） |
| **打钱弹窗** | 居中 Modal，大字号金额输入，荧光绿确认按钮 |

## 六、质量保障

### 6.1 代码规范
- 使用 ESLint + Prettier 统一风格
- 组件命名：PascalCase，文件名与组件名一致
- Git 提交遵循约定式提交（Conventional Commits）

### 6.2 测试策略
- 手动测试为主（项目规模小，个人使用）
- 关键流程回归测试清单：
  - 注册 → 登录 → 角色跳转正确
  - 管理员打钱 → 用户余额增加
  - 用户购买 → 余额扣减 → 订单生成
  - 商户上架 → 首页可见

## 七、风险与应对

| 风险 | 影响 | 应对策略 |
|------|------|----------|
| 时间不足 | 功能延期 | 优先保证核心闭环，砍掉暗黑切换等低优先级功能 |
| UI 实现偏离设计 | 视觉不统一 | 提前建好基础 UI 组件库，严格复用 |
| 后端 API 与前端对接问题 | 联调耗时 | 提前定义好 API 文档（已完备），Mock 数据先行 |
